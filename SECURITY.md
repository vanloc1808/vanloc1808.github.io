# Security Policy

## Scope

This repository hosts a personal portfolio website ([vanloc1808.github.io](https://vanloc1808.github.io)) built with Next.js and deployed as a static site on GitHub Pages. There is no versioned release cycle — the live site always reflects the latest commit on `main`.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| latest (main) | :white_check_mark: |
| older commits | :x: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do not open a public GitHub issue** for security vulnerabilities.
2. Send an email to **vanloc1808@gmail.com** with the subject line `[SECURITY] <brief description>`.
3. Include as much detail as possible:
   - A description of the vulnerability and its potential impact
   - Steps to reproduce or a proof-of-concept
   - Any suggested mitigations

### What to expect

- **Acknowledgement**: I will acknowledge receipt within **3 business days**.
- **Updates**: I will aim to provide a status update within **7 days** and a fix (if applicable) within **14 days**, depending on severity.
- **Resolution**: Once resolved, I will notify you and, if you wish, credit you in the commit message or release notes.
- **If declined**: If the report does not constitute a valid vulnerability in this project's scope, I will explain why.

## Out of Scope

The following are generally out of scope for this project:

- Vulnerabilities in third-party dependencies (please report those upstream)
- Issues in GitHub Pages infrastructure or the GitHub platform itself
- Social engineering or phishing attacks
- Denial-of-service attacks against the hosting platform

Thank you for helping keep this project safe.
